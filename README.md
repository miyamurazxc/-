## Веб-приложение для хранения, сортировки и поиска контактных данных пользователей.
Проект реализован на Java с использованием Spring Boot и демонстрирует работу с многослойной архитектурой, базой данных и системой аутентификации.  
Система позволяет вести персональную базу контактов и получать доступ к ней через браузер после авторизации.

---

## Основные возможности

- Регистрация и вход пользователя
- Безопасное хранение паролей (**BCrypt**)
- Управление контактами (**CRUD**: создание, просмотр, редактирование, удаление)
- Поиск по ключевым словам (имя, email, телефон, работа)
- Сортировка (по имени / email / дате добавления)
- Фильтры (например: только с email)
- Пагинация списка контактов
- Загрузка изображения для контакта (файл сохраняется в `static/img`, в БД хранится имя файла)
- Доступ к данным только после входа (**Spring Security**, роли)

## Структура базы данных

### Таблица `USER`
- `id` — идентификатор
- `name` — имя пользователя
- `email` — логин (уникальный)
- `password` — BCrypt-хеш пароля
- `role` — роль (например, `ROLE_USER`)
- `enabled` — активен ли пользователь
- `imageUrl` — аватар
- `about` — описание

### Таблица `CONTACT`
- `cId` — идентификатор контакта
- `name` — имя
- `secondName` — ник/фамилия
- `phone` — телефон
- `email` — email
- `work` — работа/должность
- `description` — заметки
- `image` — имя файла фото
- `createdAt` — дата добавления
- `user_id` — внешний ключ на `USER`

## Архитектура

Приложение реализует **многослойную архитектуру**, что позволяет разделить ответственность между компонентами системы и упростить её сопровождение и развитие.

### Presentation Layer
HTML-шаблоны **Thymeleaf** и стили **Bootstrap**, отвечающие за отображение пользовательского интерфейса и взаимодействие с пользователем.

### Controller Layer
Контроллеры **Spring MVC**, принимающие HTTP-запросы, обрабатывающие бизнес-логику и формирующие ответы для представления.

### Repository Layer
Репозитории **Spring Data JPA**, обеспечивающие доступ к базе данных и выполнение CRUD-операций без необходимости ручного написания SQL-запросов.

### Security Layer
Подсистема **Spring Security**, отвечающая за аутентификацию пользователей, авторизацию по ролям и защиту данных от несанкционированного доступа.

### Database Layer
Реляционная база данных **MySQL**, содержащая таблицы `USER` и `CONTACT`, связанные отношением *один-ко-многим*.

Такое архитектурное разделение обеспечивает слабую связность компонентов и упрощает сопровождение проекта.

## Как запустить проект

### Предварительные требования

- JDK  
- Maven  
- MySQL  
- IntelliJ IDEA  

### Запуск

1. Настроить подключение к базе данных в файле `application.properties`.
2. Собрать проект с помощью Maven.
3. Запустить Spring Boot-приложение.
4. Открыть браузер и перейти по адресу:  
   **http://localhost:8080**
